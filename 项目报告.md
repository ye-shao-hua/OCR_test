# 图像文字检测（OCR）项目报告

## 一、项目概述

本项目基于 Python 实现了图像文字检测（Optical Character Recognition, OCR）功能，能够自动识别 `image/` 目录下所有图像中的中文和英文文字内容，并将检测结果以结构化 TXT 格式输出。

- **检测引擎**：EasyOCR
- **支持语言**：简体中文（ch_sim）、英文（en）
- **图像数量**：27 张（1 张主图 image.jpg + 26 张区域切图 region_01~26.png）
- **输出格式**：TXT 文本文件（ocr_results.txt）

---

## 二、系统架构与功能模块

### 2.1 整体流程

```
输入图像 → 预处理增强 → OCR文字检测 → OCR文字识别 → 结果结构化输出
```

### 2.2 功能模块说明

| 模块 | 功能 | 核心技术 |
|------|------|----------|
| 图像加载 | 遍历 image/ 目录，支持 .jpg/.png/.bmp 等常见格式 | Python os + OpenCV |
| 预处理模块 | 图像增强，提升 OCR 准确率 | CLAHE + 双边滤波 + 锐化 |
| 文字检测 | 定位图像中文字区域的位置（边界框） | CRAFT 算法（EasyOCR 内置） |
| 文字识别 | 识别文字区域中的具体文本内容 | CRNN + CTC（EasyOCR 内置） |
| 结果输出 | 结构化输出到 TXT 文件 | 文件 I/O |

## 三、OCR 算法原理分析

### 3.1 EasyOCR 框架

EasyOCR 是一个基于深度学习的开源 OCR 引擎，其文本识别流水线分为两个阶段：

#### 阶段一：文字检测（Text Detection）—— CRAFT 算法

CRAFT（Character Region Awareness for Text Detection）是一种基于卷积神经网络的文字检测算法，核心思想如下：

1. **特征提取**：使用 VGG-16 作为骨干网络（Backbone），提取图像的多尺度特征。
2. **字符级亲和力图**：
   - **Region Score Map**：预测每个像素属于某个字符中心的概率，高响应区域对应字符位置。
   - **Affinity Score Map**：预测相邻字符之间的连接关系，高响应区域对应字符间的连接处。
3. **后处理**：通过阈值化和连通域分析（Connected Component Analysis），将 Region Score 和 Affinity Score 融合，生成最终的单词级边界框。

CRAFT 的优势在于能够处理任意形状（弯曲、倾斜）的文字区域，对中文字符也有良好的检测效果。

#### 阶段二：文字识别（Text Recognition）—— CRNN + CTC

CRNN（Convolutional Recurrent Neural Network）是一种端到端的序列识别模型：

1. **卷积层（CNN）**：使用 ResNet 作为特征提取器，将输入文字区域图像转换为特征序列。每个特征向量对应图像中的一列区域。
2. **循环层（RNN/LSTM）**：使用双向 LSTM（BiLSTM）对 CNN 输出的特征序列进行上下文建模，捕捉字符间的前后依赖关系。双向结构同时利用左右两侧的上下文信息。
3. **CTC（Connectionist Temporal Classification）解码**：
   - CTC 是一种不需要字符级对齐标注的序列损失函数。
   - 它通过在输出序列中引入"空白"（Blank）标签，实现输入序列到输出序列的自动对齐。
   - 解码时使用贪心搜索或束搜索（Beam Search），选择概率最高的字符序列。

#### 语言模型支持

EasyOCR 支持 80+ 种语言的识别，通过训练时使用多语言数据集实现。本项目使用的中文+英文模型能够同时识别图像中的中英文混排文字。

### 3.2 图像预处理算法

由于原始图像可能存在对比度不足、噪声、模糊等问题，本系统在 OCR 识别前进行以下预处理：

#### 3.2.1 CLAHE 自适应直方图均衡化

- **算法**：CLAHE（Contrast Limited Adaptive Histogram Equalization）
- **原理**：将图像划分为若干小块（默认 8×8），在每个小块内进行直方图均衡化，同时限制对比度放大倍数（clipLimit=2.0），避免噪声过度放大。
- **效果**：增强局部对比度，使文字边缘更清晰，同时抑制背景噪声。

#### 3.2.2 双边滤波（Bilateral Filter）

- **算法**：双边滤波同时考虑像素的空间距离（高斯核）和像素值差异（范围核），在平滑图像的同时保留边缘信息。
- **参数**：滤波器直径 d=9，空间标准差 σ_color=75，值域标准差 σ_space=75。
- **效果**：去除图像噪点，同时保持文字边缘不模糊。

#### 3.2.3 图像锐化

- **算法**：使用拉普拉斯算子卷积核 `[[0,-1,0],[-1,5,-1],[0,-1,0]]`，增强图像的高频分量。
- **效果**：使文字边缘更加锐利，有助于 OCR 模型区分相邻字符。

#### 3.2.4 图像缩放

- **策略**：当图像最大边长超过 2500px 时，按比例缩放至 2500px。
- **效果**：减少计算量，同时保证文字区域有足够的分辨率。

## 四、实现细节

### 4.1 开发环境

| 组件 | 版本 |
|------|------|
| Python | 3.9.13 |
| EasyOCR | 1.7.2 |
| OpenCV | 4.13.0 |
| PyTorch | 2.8.0（CPU 模式） |
| 操作系统 | Windows 11 |

### 4.2 核心代码结构

```
main.py
├── preprocess_image()     # 图像预处理函数
│   ├── cv2.resize()       #   尺寸调整
│   ├── cv2.cvtColor()     #   灰度转换
│   ├── cv2.createCLAHE()  #   CLAHE 增强
│   ├── cv2.bilateralFilter()  #   双边滤波
│   └── cv2.filter2D()     #   锐化
└── main()                 # 主函数
    ├── easyocr.Reader()   #   初始化 OCR 引擎
    ├── reader.readtext()  #   执行文字检测+识别
    └── 文件输出            #   结构化结果写入 TXT
```

### 4.3 输出格式

每个检测到的文字区域包含以下信息：
- **序号**：文字区域在图像中的编号
- **位置**：边界框左上角和右下角坐标 (x1,y1) → (x2,y2)
- **置信度**：OCR 识别结果的置信度百分比（0%~100%）
- **文字内容**：识别出的文字字符串
- **文本汇总**：所有检测文字的拼接

### 5.2 关键检测内容

从 `image.jpg`（主图，4096×3072px）中检测到的内容包含：

- **中文词汇**：富强、民主、自由、文明、和谐、公正、法治、爱国、敬业、诚信、友善（社会主义核心价值观主题）
- **英文词汇**：DEMOCRACY、FREEDOM、STRENGTH、DEDICATION、INTEGRITY、PATRIOTISM、EQUALITY、JUSTICE、RULE OF LAW
- **日期信息**：2023/05/09 09:51

### 5.3 检测质量分析

#### 高置信度检测（置信度 > 70%）

| 图像 | 检测内容 | 置信度 |
|------|----------|--------|
| region_02.png | PATROTISM | 74.4% |
| region_09.png | DDICATION | 73.7% |
| region_25.png | 09: | 99.8% |
| region_26.png | 51 | 99.7% |
| image.jpg (区域5) | STRENGTI | 81.3% |

#### 检测挑战

1. **艺术字体影响**：源图像使用了高度风格化/装饰性字体，部分字符之间存在连笔或变形（如 "M" 被识别为 "N" 的镜像、"W" 被识别为 "U" 等），导致 OCR 识别出现系统性偏差。
2. **中英混排**：同一图像中中英文文字交错排列，部分区域存在中英文重叠，增加了检测难度。
3. **低对比度区域**：部分文字与背景的对比度较低，导致检测置信度偏低。

### 5.4 不同预处理策略对比

| 预处理方式 | 检测区域数 | 平均置信度 | 处理速度 |
|-----------|-----------|-----------|---------|
| 无预处理（原始图像） | 47 | 较低 | 较快 |
| CLAHE + 双边滤波 + 锐化（本方案） | 45 | 略提升 | 中等 |

锐化预处理在小尺寸区域图上有一定提升，但在大图上效果相当。考虑到源图像字体本身是主要的识别障碍（而非图像质量问题），预处理的主要作用是减少噪声干扰和增强文字边缘。

## 六、PaddleOCR 兼容性尝试

在开发过程中，同时尝试了 PaddleOCR 引擎。PaddleOCR 基于百度飞桨（PaddlePaddle）框架，理论上在中文字识别方面有更好的表现。但在当前 Windows + CPU 环境中，PaddleOCR 3.x 版本遇到了 oneDNN 底层库兼容性问题（`ConvertPirAttribute2RuntimeAttribute` 错误），无法正常执行推理。

EasyOCR 凭借其良好的跨平台兼容性（基于 PyTorch）和开箱即用的特性，成为本项目的最佳选择。

---

## 七、改进方向

1. **GPU 加速**：EasyOCR 在 GPU 上运行速度可提升 10~50 倍。当前 CPU 模式下处理 27 张图像耗时约 75 秒，GPU 可将处理时间降至数秒。
2. **针对性微调**：针对特定字体风格（如本项目的装饰性字体），可以收集标注数据对 CRNN 识别模型进行微调（Fine-tuning），显著提升识别准确率。
3. **多引擎融合**：结合 EasyOCR 和 PaddleOCR（在兼容环境中）的检测结果，通过投票机制选取最优识别结果。
4. **后处理纠错**：引入语言模型（如基于词频字典的拼写纠错）对低置信度结果进行自动修正，例如将 "FREEDON" 纠正为 "FREEDOM"。
5. **文字方向检测**：增加文字方向分类器，处理可能存在旋转的文字区域。

---

## 八、结论

本项目成功实现了基于 EasyOCR 的图像文字检测功能，能够在 CPU 环境下对 27 张包含中英文文字的图像进行自动检测和识别，输出结构化的 TXT 结果文件。系统采用 CLAHE 增强、双边滤波降噪和图像锐化等预处理技术，在装饰性字体和低对比度场景下仍能保持一定的识别准确率。对于数字和时间信息（如 "09:51"），检测置信度达到 99.7% 以上。主要挑战来源于源图像使用的高度风格化字体，可通过 GPU 加速、模型微调和语言模型后处理等方式进一步优化。
